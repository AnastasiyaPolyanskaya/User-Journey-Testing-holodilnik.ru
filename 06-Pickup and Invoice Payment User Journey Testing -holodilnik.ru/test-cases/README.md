## Тест-кейсы

В данном разделе представлены тест-кейсы, разработанные для проверки управления неоплаченными заказами и корректности отображения состояния раздела «Избранное» в мобильной версии сайта holodilnik.ru.

Тестирование направлено на выявление функциональных и UI-дефектов, возникающих после оформления заказа и при работе с пользовательскими данными.

## Описание протестированных сценариев

В рамках тест-кейсов проверяются следующие сценарии:

- создание неоплаченного заказа;
- переход в раздел «Заказы» после оформления;
- наличие кнопки «Оплатить заказ»;
- наличие кнопки удаления неоплаченного заказа;
- корректность отображения состояния заказа;
- добавление товара в «Избранное»;
- удаление товара из «Избранного»;
- корректность отображения числового индикатора в разделе «Избранное».

## Типы тестирования

Тест-кейсы включают:

- функциональное тестирование;
- тестирование пользовательских состояний (state testing);
- UI/UX-тестирование;
- позитивные сценарии;
- негативные сценарии.

## Структура тест-кейсов

Каждый тест-кейс содержит:

- идентификатор тест-кейса;
- предусловия;
- шаги воспроизведения;
- ожидаемый результат;
- фактический результат;
- информацию об устройстве и версии браузера;
- статус выполнения (Passed / Failed).

## Окружение тестирования

- Устройство: iPhone 14 Pro (iOS 18.6.2)
- Браузер: Яндекс Браузер
- Версия браузера: 25.12.5
- Платформа: Mobile Web
