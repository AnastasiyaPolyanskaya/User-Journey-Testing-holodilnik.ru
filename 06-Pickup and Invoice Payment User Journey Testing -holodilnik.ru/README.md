# 06-Pickup and Invoice Payment User Journey Testing — holodilnik.ru

Практический учебный проект по ручному тестированию пользовательского пути оформления заказа с самовывозом из пункта выдачи заказов (ПВЗ) и оплатой по счёту в интернет-магазине holodilnik.ru.

Проект направлен на проверку корректности альтернативного сценария оформления заказа, отличного от стандартной оплаты картой, а также на выявление UX- и функциональных проблем на этапах оформления и взаимодействия с интерфейсом.

## Цель проекта

Проверить стабильность пользовательского пути оформления заказа с самовывозом из ПВЗ и оплатой по счёту, а также выявить проблемы интерфейса и управления заказом, влияющие на пользовательский опыт.

## Объект тестирования

Интернет-магазин holodilnik.ru  
Платформа: Mobile Web  
Устройство: iPhone 14 Pro (iOS 18.6.2)  
Браузер: Яндекс Браузер  
Тип тестирования: Manual QA

## Описание протестированных сценариев

В рамках проекта выполнено тестирование следующих пользовательских сценариев:

- переход из главной страницы в каталог и листинг товаров;
- добавление товара в корзину;
- оформление заказа с авторизацией при оформлении;
- выбор способа доставки «Самовывоз из ПВЗ»;
- ввод валидных и невалидных данных;
- проверка отображения сообщений об ошибках;
- выбор способа оплаты «Оплата по счёту»;
- завершение оформления заказа;
- проверка управления заказом и состояния интерфейса;
- проверка отображения раздела «Избранное» при отсутствии товаров.

## Типы тестирования

В проекте использовались следующие типы тестирования:

- функциональное тестирование;
- E2E-тестирование пользовательского пути;
- UI/UX-тестирование;
- тестирование альтернативных способов оплаты;
- позитивные и негативные сценарии.

## Артефакты проекта

В репозитории представлены:

- отчёт о прохождении пользовательского пути;
- тест-кейсы;
- баг-репорты;
- скриншоты и видеозаписи, фиксирующие выявленные дефекты;
- дополнительные видеоматериалы с воспроизведением проблемных сценариев.

## Результаты

В ходе тестирования выявлено:

- корректная работа авторизации и ввода данных;
- корректное отображение сообщений об ошибках при невалидном вводе;
- стабильный переход к оплате по счёту;
- отсутствие кнопки удаления заказа;
- отображение числового индикатора в разделе «Избранное» при отсутствии товаров.

Выявленные дефекты негативно влияют на удобство управления заказом и пользовательский опыт при использовании альтернативных сценариев оформления заказа.
