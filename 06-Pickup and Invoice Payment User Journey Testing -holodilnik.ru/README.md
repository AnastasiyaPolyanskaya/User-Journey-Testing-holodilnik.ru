# 06-Order Management and Favorites State Testing - holodilnik.ru

Практический учебный проект по ручному тестированию управления заказами и состояния пользовательских данных в мобильной версии интернет-магазина holodilnik.ru.

Проект направлен на проверку поведения системы после оформления заказа, работы раздела «Заказы», а также корректности отображения состояния раздела «Избранное».

## Цель проекта

Проверить корректность управления неоплаченными заказами и отображения пользовательских данных в мобильной версии сайта, а также выявить функциональные и UI-дефекты, влияющие на пользовательский опыт.

## Объект тестирования

Интернет-магазин holodilnik.ru  
Платформа: Mobile Web  
Устройство: iPhone 14 Pro (iOS 18.6.2)  
Браузер: Яндекс Браузер  
Тип тестирования: Manual QA

## Описание протестированных сценариев

В рамках проекта протестированы следующие сценарии:

- оформление заказа с авторизацией при оформлении;
- переход в раздел «Заказы» после создания неоплаченного заказа;
- проверка наличия кнопки «Оплатить заказ»;
- проверка наличия кнопки удаления неоплаченного заказа;
- проверка корректности отображения состояния заказа;
- добавление и удаление товара из «Избранного»;
- проверка корректности числового индикатора в разделе «Избранное».

## Типы тестирования

В проекте использованы:

- функциональное тестирование;
- тестирование пользовательских состояний;
- UI/UX-тестирование;
- позитивные и негативные сценарии;
- тестирование пользовательского пути.

## Выявленные проблемы

В ходе тестирования зафиксированы:

- отсутствие кнопки удаления неоплаченного заказа;
- некорректное отображение числового индикатора в разделе «Избранное»;
- корректная работа кнопки оплаты для неоплаченного заказа.

Выявленные дефекты влияют на удобство управления заказами и создают неконсистентное поведение пользовательского интерфейса.
