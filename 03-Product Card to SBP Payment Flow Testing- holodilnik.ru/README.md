# 03-Product Card to SBP Payment Flow Testing — holodilnik.ru

Практический учебный проект по ручному тестированию пользовательского пути оплаты через СБП при оформлении заказа из карточки товара интернет-магазина holodilnik.ru.

Проект направлен на проверку корректности перехода к оплате через СБП, доступности способа оплаты и стабильности пользовательского сценария при выборе оплаты напрямую из карточки товара.

## Цель проекта

Проверить корректность работы способа оплаты через СБП при оформлении заказа из карточки товара, а также выявить функциональные и UX-проблемы, влияющие на возможность завершения покупки.

## Объект тестирования

Интернет-магазин holodilnik.ru  
Платформа: Web  
Тип тестирования: Manual QA

## Описание протестированных сценариев

В рамках проекта выполнено тестирование следующих пользовательских сценариев:

- переход из карточки товара к оформлению заказа;
- доступность способа оплаты через СБП;
- выбор способа оплаты СБП;
- корректность перехода к оплате через СБП;
- поведение системы при отсутствии или недоступности СБП;
- проверка пользовательского пути при альтернативных способах оплаты;
- обработка ошибок и уведомлений, связанных с выбором способа оплаты.

## Типы тестирования

В проекте использовались следующие типы тестирования:

- функциональное тестирование;
- тестирование пользовательского пути;
- UI/UX-тестирование элементов выбора оплаты;
- позитивные и негативные сценарии.

## Артефакты проекта

В репозитории представлены:

- тест-кейсы;
- баг-репорты;
- скриншоты с фиксацией поведения интерфейса и выявленных дефектов;
- результаты прохождения пользовательских сценариев.

## Формат тест-кейсов

Тест-кейсы оформлены в табличном виде и содержат:

- идентификатор тест-кейса;
- описание сценария;
- предусловия;
- шаги воспроизведения;
- ожидаемый и фактический результат;
- информацию об устройстве и версии браузера;
- статус выполнения (Passed / Failed).

## Результаты

В ходе тестирования выявлены:

- особенности и ограничения выбора оплаты через СБП;
- UX-проблемы, связанные с доступностью способа оплаты;
- корректные сценарии перехода к оплате при использовании альтернативных способов оплаты.
