## Тест-кейсы

В данном разделе представлены тест-кейсы, разработанные для проверки пользовательского пути оплаты через СБП при оформлении заказа из карточки товара интернет-магазина holodilnik.ru.

Тест-кейсы направлены на проверку доступности способа оплаты СБП, корректности отображения элемента оплаты и возможности завершения пользовательского пути при выборе данного способа оплаты.

## Описание тестируемых сценариев

В рамках тест-кейсов проверяются следующие сценарии:

- переход из карточки товара в корзину и оформление заказа;
- доступность способа оплаты «СБП» на этапе оформления;
- корректность отображения способа оплаты СБП;
- поведение системы при отсутствии способа оплаты СБП;
- влияние недоступности СБП на возможность завершения оформления заказа;
- проверка альтернативных способов оплаты при отсутствии СБП.

## Типы тестирования

Тест-кейсы включают следующие типы тестирования:

- функциональное тестирование;
- тестирование пользовательского пути;
- тестирование способов оплаты;
- позитивные и негативные сценарии;
- UI/UX-тестирование элементов выбора оплаты.

## Структура тест-кейсов

Каждый тест-кейс содержит:

- идентификатор тест-кейса;
- описание сценария;
- предусловия;
- шаги воспроизведения;
- тестовые данные;
- ожидаемый результат;
- фактический результат;
- информацию об устройстве и версии браузера;
- статус выполнения (Passed / Failed).

## Формат хранения

Тест-кейсы оформлены в табличном виде и предназначены для фиксации результатов ручного тестирования пользовательского пути оплаты через СБП.
