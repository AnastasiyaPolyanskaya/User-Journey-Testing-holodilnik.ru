## Тест-кейсы

В данном разделе представлены тест-кейсы, разработанные для проверки пользовательского интерфейса и сохранности данных в интернет-магазине holodilnik.ru.

Тест-кейсы направлены на оценку визуального отображения страниц, поведения интерфейсных элементов, а также корректности сохранения пользовательских данных при выходе и повторной авторизации.

## Описание тестируемых сценариев

В рамках тест-кейсов проверяются следующие сценарии:

- визуальное отображение главной страницы и листинга товаров;
- корректность UI-элементов при просмотре каталога;
- наличие и влияние отвлекающих элементов интерфейса;
- сохранение данных доставки после выхода из системы;
- корректность восстановления данных при повторном входе;
- стабильность пользовательской сессии при оформлении заказа.

## Типы тестирования

Тест-кейсы включают следующие типы тестирования:

- функциональное тестирование;
- UI/UX-тестирование;
- тестирование сохранности данных;
- позитивные сценарии;
- негативные сценарии.

## Структура тест-кейсов

Каждый тест-кейс содержит:

- идентификатор тест-кейса;
- краткое описание сценария;
- предусловия;
- шаги воспроизведения;
- ожидаемый результат;
- фактический результат;
- информацию об устройстве и версии браузера;
- статус выполнения (Passed / Failed).

## Формат хранения

Тест-кейсы оформлены в табличном виде и предназначены для фиксации результатов ручного тестирования.
