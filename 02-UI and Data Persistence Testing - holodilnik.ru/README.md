# 02-UI and Data Persistence Testing — holodilnik.ru

Практический учебный проект по ручному тестированию пользовательского интерфейса и сохранности данных интернет-магазина holodilnik.ru.

Проект основан на тест-кейсах, направленных на проверку визуального отображения страниц сайта, а также корректности сохранения пользовательских данных при выходе и повторном входе в систему.

## Цель проекта

Проверить корректность отображения UI-элементов на основных пользовательских экранах и стабильность сохранения данных доставки при повторной авторизации пользователя.

## Объект тестирования

Интернет-магазин holodilnik.ru  
Платформа: Web  
Тип тестирования: Manual QA

## Описание протестированных сценариев

В рамках проекта выполнено тестирование следующих сценариев:

- проверка UI главной страницы и листинга товаров при авторизованном пользователе;
- анализ визуальных элементов, влияющих на пользовательский опыт;
- выявление отвлекающих и перекрывающих элементов интерфейса;
- проверка сохранения данных доставки после выхода и повторного входа в систему;
- проверка корректности работы пользовательской сессии;
- проверка влияния повторной авторизации на данные оформления заказа.

## Типы тестирования

В проекте использовались следующие типы тестирования:

- функциональное тестирование;
- UI/UX-тестирование;
- тестирование сохранности данных;
- позитивные сценарии;
- негативные сценарии.

## Артефакты проекта

В репозитории представлены:

- тест-кейсы;
- баг-репорты;
- скриншоты с фиксацией выявленных дефектов интерфейса и поведения системы;
- результаты выполнения тестов.

## Формат тест-кейсов

Тест-кейсы оформлены в табличном виде и содержат:

- идентификатор тест-кейса;
- описание сценария;
- шаги воспроизведения;
- ожидаемый и фактический результат;
- информацию об устройстве и версии браузера;
- статус выполнения (Passed / Failed).

## Результаты

В ходе тестирования выявлены:

- UX-проблемы, связанные с визуально отвлекающими элементами интерфейса;
- потеря данных доставки после повторной авторизации пользователя;
- корректная работа основных пользовательских сценариев при отсутствии выхода из системы.
