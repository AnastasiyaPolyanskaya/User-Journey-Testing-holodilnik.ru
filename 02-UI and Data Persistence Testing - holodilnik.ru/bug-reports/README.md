## Баг-репорты

В данном разделе представлены баг-репорты, выявленные в ходе тестирования пользовательского интерфейса и сохранности данных интернет-магазина holodilnik.ru.

Баги были обнаружены при проверке визуального отображения страниц, поведения UI-элементов, а также при анализе корректности сохранения пользовательских данных при выходе и повторной авторизации.

## Описание выявленных дефектов

В рамках проекта зафиксированы следующие типы дефектов:

- визуально отвлекающие элементы интерфейса, влияющие на пользовательский опыт;
- некорректное отображение UI-элементов в листинге товаров;
- потеря данных доставки после выхода и повторного входа в систему;
- ошибки, связанные с сохранностью пользовательской сессии.

## Структура баг-репортов

Каждый баг-репорт содержит:

- уникальный идентификатор;
- краткое описание дефекта;
- окружение (устройство, версия ОС, браузер);
- предусловия;
- шаги воспроизведения;
- ожидаемый результат;
- фактический результат;
- серьёзность (Severity);
- приоритет (Priority);
- комментарий с описанием влияния дефекта.

## Классификация дефектов

В проекте используются следующие уровни:

- **Severity:** Minor, Major  
- **Priority:** Medium, High  

Классификация дефектов выполнена на основе влияния проблемы на пользовательский опыт и возможность завершения ключевых сценариев.

## Формат хранения

Баг-репорты оформлены в текстовом формате и предназначены для детального описания и анализа выявленных дефектов.
