# 01-Checkout and Delivery User Journey Testing — holodilnik.ru

Практический учебный проект по ручному тестированию пользовательских путей интернет-магазина holodilnik.ru.

Проект основан на тест-кейсах сквозного прохождения пользовательского сценария — от выбора товара в каталоге до перехода на страницу оплаты банковской картой, с проверкой авторизации, доставки и валидации данных.

## Цель проекта

Проверить корректность и стабильность пользовательского пути оформления заказа, а также выявить функциональные и UI/UX-проблемы на этапах авторизации, доставки и оплаты.

## Объект тестирования

Интернет-магазин holodilnik.ru  
Платформа: Web  
Тип тестирования: Manual QA

## Описание протестированных сценариев

В рамках проекта выполнено тестирование следующих пользовательских сценариев:

- получение кода авторизации при оформлении заказа;
- ввод валидных и невалидных данных в разделе «Доставка»;
- проверка обязательных и необязательных полей (квартира, подъезд, этаж);
- проверка обработки ошибок при вводе некорректных данных;
- выбор даты и времени доставки;
- выбор способа оплаты (банковская карта);
- переход на страницу оплаты картой;
- проверка сохранности данных при переходах;
- проверка UI-анимаций и выпадающих списков в листинге.

Тестирование включает как позитивные, так и негативные сценарии.

## Артефакты проекта

В репозитории представлены:
- тест-кейсы;
- баг-репорты;
- скриншоты с фиксацией обнаруженных дефектов и особенностей интерфейса;
- результаты прохождения пользовательских путей.

## Формат тест-кейсов

Тест-кейсы оформлены в табличном виде и содержат:

- идентификатор тест-кейса;
- описание сценария и шаги воспроизведения;
- тестовые данные;
- ожидаемый и фактический результат;
- информацию об устройстве и версии браузера;
- статус выполнения (Passed / Failed).

## Результаты

В ходе тестирования выявлены:
- ошибки в валидации адресных данных;
- некорректная обработка необязательных полей доставки;
- проблемы в UI-анимации выпадающих списков;
- успешные сценарии прохождения пути до оплаты картой
