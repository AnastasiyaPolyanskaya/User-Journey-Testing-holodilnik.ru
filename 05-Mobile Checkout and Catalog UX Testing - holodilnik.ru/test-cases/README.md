## Тест-кейсы

В данном разделе представлены тест-кейсы, разработанные для проверки навигации и структуры каталога в мобильной версии интернет-магазина holodilnik.ru.

Тест-кейсы направлены на выявление UX- и функциональных проблем, связанных с переходами между листингом и карточкой товара, а также доступностью категорий товаров на мобильных устройствах.

## Описание тестируемых сценариев

В рамках тест-кейсов проверяются следующие пользовательские сценарии:

- переход из главной страницы в листинг товаров;
- выбор категории товаров из листинга;
- переход из листинга в карточку товара;
- наличие кнопки возврата из карточки товара в категорию;
- возврат пользователя к списку товаров после просмотра карточки;
- наличие прямого доступа к категориям товаров в основном листинге;
- доступность категории товаров «Часы» без перехода через другие разделы.

## Типы тестирования

Тест-кейсы включают следующие типы тестирования:

- функциональное тестирование;
- UI/UX-тестирование;
- тестирование навигации;
- негативные сценарии.

## Структура тест-кейсов

Каждый тест-кейс содержит:

- идентификатор тест-кейса;
- описание сценария;
- предусловия;
- шаги воспроизведения;
- ожидаемый результат;
- фактический результат;
- информацию об устройстве и версии браузера;
- статус выполнения (Passed / Failed).

## Формат хранения

Тест-кейсы оформлены в табличном виде и используются для фиксации результатов ручного тестирования навигации и пользовательского опыта на мобильной версии сайта.
