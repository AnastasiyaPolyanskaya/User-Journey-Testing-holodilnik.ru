# 05-Mobile Checkout and Catalog UX Testing — holodilnik.ru

Практический учебный проект по ручному тестированию пользовательского пути и пользовательского опыта на мобильной версии интернет-магазина holodilnik.ru.

Проект направлен на проверку корректности прохождения сквозного пользовательского сценария от главной страницы до оплаты картой, а также на выявление UX- и навигационных проблем в каталоге и листинге товаров на мобильных устройствах.

## Цель проекта

Проверить стабильность и удобство пользовательского пути оформления заказа на мобильной версии сайта, а также выявить проблемы навигации и структуры каталога, влияющие на пользовательский опыт.

## Объект тестирования

Интернет-магазин holodilnik.ru  
Платформа: Mobile Web  
Устройство: iPhone 14 Pro (iOS 18.6.2)  
Браузер: Яндекс Браузер  
Тип тестирования: Manual QA

## Описание протестированных сценариев

В рамках проекта выполнено тестирование следующих пользовательских сценариев:

- переход с главной страницы в каталог и листинг товаров;
- работа фильтров по моделям и характеристикам;
- переход из листинга в карточку товара;
- добавление товара в корзину;
- оформление заказа;
- авторизация пользователя;
- ввод валидных и невалидных данных;
- проверка отображения сообщений об ошибках;
- переход к оплате банковской картой;
- анализ навигации и доступности категорий в каталоге.

## Типы тестирования

В проекте использовались следующие типы тестирования:

- функциональное тестирование;
- E2E-тестирование пользовательского пути;
- UI/UX-тестирование;
- позитивные и негативные сценарии.

## Артефакты проекта

В репозитории представлены:

- отчёт о прохождении пользовательского пути;
- скриншоты и видеозапись тестирования;
- описание позитивных и негативных UX-наблюдений;
- результаты выполнения сценариев.

## Результаты

В ходе тестирования выявлено:

- корректная работа фильтров по моделям и характеристикам;
- корректная обработка валидных и невалидных данных с отображением сообщений об ошибках;
- стабильный переход к оплате банковской картой;
- отсутствие кнопки возврата назад при переходе из листинга в карточку товара;
- отсутствие прямого доступа к части категорий на основном листинге товаров, что усложняет навигацию по каталогу на мобильной версии сайта.
