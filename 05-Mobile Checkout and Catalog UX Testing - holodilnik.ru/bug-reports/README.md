## Баг-репорты

В данном разделе представлены баг-репорты, выявленные в ходе тестирования навигации и структуры каталога в мобильной версии интернет-магазина holodilnik.ru.

Дефекты обнаружены при проверке пользовательских сценариев перехода между листингом и карточкой товара, а также доступности категорий товаров в основном каталоге.

## Описание выявленных дефектов

В рамках проекта зафиксированы следующие дефекты:

- отсутствие кнопки возврата из карточки товара в категорию товаров;
- отсутствие прямого доступа к категории товаров «Часы» в основном листинге;
- нарушение логики навигации при просмотре каталога на мобильной версии сайта.

Выявленные проблемы увеличивают количество действий, необходимых пользователю для поиска товаров, и негативно влияют на пользовательский опыт.

## Структура баг-репортов

Каждый баг-репорт содержит:

- уникальный идентификатор и название дефекта;
- окружение (устройство, версия ОС, браузер);
- предусловия;
- шаги воспроизведения;
- ожидаемый результат;
- фактический результат;
- серьёзность (Severity);
- приоритет (Priority);
- комментарий с описанием влияния дефекта.

## Классификация дефектов

В проекте используются следующие уровни:

- **Severity:** Major  
- **Priority:** High  

Классификация выполнена с учётом влияния дефектов на пользовательский путь и удобство навигации.

## Формат хранения

Баг-репорты оформлены в текстовом виде и предназначены для фиксации и анализа дефектов, выявленных в процессе ручного тестирования мобильной версии сайта.
